üéØ System Overview

The Greenhouse Control System is a distributed IoT solution consisting of three main components:

üîß ESP32 Greenhouse Controller

Embedded device managing sensors, relays, and real-time automation:

  • Temperature sensors (DS18B20)
  • Current monitoring (ACS712)
  • Relay control (up to 15 channels)
  • Routine automation engine
  • Alert system (WhatsApp/Telegram)

üåê Raspberry Pi Proxy

Central hub for web UI, weather caching, and device management:

  • Express.js web server
  • WebSocket fan-out (one ESP32 ‚Üí many browsers)
  • Weather API caching (OpenMeteo)
  • Device configuration management
  • OTA firmware updates

‚öôÔ∏è ESP32 Firmware Architecture

Core Systems

1. Device Manager

Manages all connected sensors and relays with persistent storage.

  • DS18B20 temperature sensors (OneWire bus)
  • ACS712 current sensor (ADC)
  • Digital relay control outputs
  • Configuration saved to NVS (Non-Volatile Storage)

2. Routine Engine

Autonomous automation based on triggers:

Trigger Types:
- TIME: Execute at specific time or interval
- TEMPERATURE: When temp crosses threshold
- CURRENT: When device current changes
- MANUAL: User-triggered via UI

3. Weather System

üîÑ Dynamic Behavior:
  • When Pi connected: Weather checking PAUSED ‚è∏Ô∏è
  • When Pi disconnected: Weather checking RESUMED ‚èØÔ∏è
  • Update interval: 30 minutes (when active)
  • API source: OpenMeteo (free, no auth required)

4. Alert System

Sends notifications via WhatsApp/Telegram when conditions are met:

  • Relay state changes with confirmation
  • Temperature threshold violations
  • System errors and warnings
  • Daily status reports (optional)

Task Structure (FreeRTOS)

Task Name Core Priority Interval Purpose
SyncTask 0 1 500ms WebSocket broadcast of telemetry
TempTask 0 2 10s DS18B20 sensor reading
CurrentTask 0 2 1s ACS712 current sampling
ProxyConnCheck 1 1 1s Monitor Pi connection (ping timeout)

Memory Management

  • Static buffers for fixed-size data (IP addresses, coordinates)
  • JSON documents auto-sized (ArduinoJson 7)
  • Heap fragmentation minimized with careful allocation
  • Watchdog timer prevents freezing

üåê Raspberry Pi Proxy Server

Node.js Express application that acts as the central hub for the system.

Key Features

  • WebSocket Proxy: Maintains single connection to ESP32, fans out to multiple browsers
  • Weather Caching: Fetches from OpenMeteo, caches for 10 minutes
  • Static File Serving: Serves HTML/CSS/JS files (faster than ESP32 LittleFS)
  • Device Management: Configuration UI and REST API
  • OTA Updates: Upload new firmware for ESP32 and Pi

üå¶Ô∏è Weather Management System

Architecture

The weather system dynamically switches between local ESP32 fetching and Pi-based caching based on connectivity:

When Pi Proxy is CONNECTED (‚è∏Ô∏è Paused)

  • ESP32 pauses ALL local weather fetching
  • ESP32 saves CPU cycles and HTTP quota
  • Pi proxy caches weather every 10 minutes
  • UI requests weather from /api/weather on Pi

When Pi Proxy is DISCONNECTED (‚èØÔ∏è Resumed)

  • ESP32 automatically resumes weather fetching
  • Updates every 30 minutes from OpenMeteo
  • UI displays weather from local ESP32 cache
  • No external dependencies needed

Weather Data Sources

OpenMeteo API Open-Meteo (Free) No Global, free tier

üîß Complete Setup Guide

Prerequisites

  • ESP32 with PlatformIO installed
  • Raspberry Pi (any model with Node.js support)
  • WiFi network accessible from both devices
  • SSH access to Pi

Step 1: Configure ESP32

  1. Update include/Secrets.h: 
    const char DEFAULT_SSID[] = "YourWiFi";
const char DEFAULT_PASS[] = "password";
  2. Build and upload: 
    cd 260125-185311-esp32dev
platformio run --target upload
  3. Note ESP32 IP address from serial output

Step 2: Deploy Pi Proxy

  1. Navigate to deploy script: 
    cd raspberry-pi/greenhouse-proxy
  2. Run deployment: 
    ./deploy.sh nkepah@100.92.151.67 --esp32-ip=10.0.0.163
  3. Service starts automatically

Step 3: Install Nginx Reverse Proxy

  1. SSH to Pi: 
    ssh nkepah@100.92.151.67
  2. Install and enable Nginx: 
    sudo apt update && sudo apt install -y nginx
  3. Copy config: 
    sudo cp /tmp/nginx-greenhouse.conf /etc/nginx/sites-available/greenhouse-proxy
sudo ln -sf /etc/nginx/sites-available/greenhouse-proxy /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo systemctl restart nginx
  4. Verify: 
    curl http://100.92.151.67/

Step 4: Configure Location & Weather

  1. Visit ESP32 setup page: http://{esp32-ip}/setup
  2. Enter location coordinates (latitude/longitude)
  3. Set timezone (GMT offset)
  4. Test weather fetch

Step 5: First Run Verification

‚úì Checklist
  • [ ] ESP32 connected to WiFi (check serial output)
  • [ ] Pi proxy running (sudo systemctl status greenhouse-proxy)
  • [ ] WebSocket connection established (green indicator in UI)
  • [ ] Weather data loading (visible in dashboard)
  • [ ] Nginx serving on port 80 (curl http://100.92.151.67/)

Troubleshooting

üîç Common Issues

No WebSocket Connection

  • Check firewall allows port 3000
  • Verify ESP32 can reach Pi on configured IP
  • Check Pi proxy logs: sudo journalctl -u greenhouse-proxy

Weather Not Updating

  • Check location coordinates are set
  • Verify internet connectivity on Pi
  • Check weather cache in Pi proxy logs

UI Shows Old Data

  • Clear browser cache (Ctrl+Shift+Del)
  • Check WebSocket connection status
  • Reload page and check network tab
